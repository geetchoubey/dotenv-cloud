Configuration file — dotenv-cloud.toml

TOML is the config format. When --config / DOTENV_CLOUD_CONFIG aren't set, dotenv-cloud searches the current directory and walks up parents for the first file named dotenv-cloud.toml. Unknown keys are rejected (typos surface in validate) — except the free-form [providers.*] tables, which are passed through to plugins. Every section is optional.

Top level & profiles

KeyTypeDefaultDescription
default_profilestring"dev"Profile used when neither --profile nor DOTENV_CLOUD_PROFILE is set.
[profile.<name>] · env_filestring".env"Primary dotenv file for that profile.
[profile.<name>] · env_local_filestring".env.local"Gitignored local-override file for that profile.
default_profile = "dev"

[profile.prod]
env_file = ".env.production"
env_local_file = ".env.production.local"

[defaults]

Static fallback values (lowest precedence by default). Free-form string → string pairs.

[defaults]
LOG_LEVEL = "info"
PORT = "3000"

[precedence]

KeyTypeDefaultDescription
orderarray of string["cli","system","remote","env.local","env","defaults"]Sources highest-precedence first. No duplicates. Omitted sources keep their default relative order.

Allowed source identifiers:

IdentifierSource
cliValues from --set KEY=VALUE
systemVariables already in the process environment
remoteValues resolved from secret references (aws-secrets://, …)
env.localThe profile's env_local_file
envThe profile's env_file
defaultsThe [defaults] table

[providers.*]

Per-provider configuration, passed straight through to the plugin as its provider_config during resolution. The core does not validate the keys inside these tables — they're defined by each provider. The keys below are what the bundled AWS provider understands (examples, not a fixed schema).

# AWS provider — schemes aws-secrets, aws-ssm
[providers.aws]
region = "us-east-1"
profile = "default"      # named profile from ~/.aws (omit for the default chain)
timeout_ms = 2000

[providers.aws.ssm]
with_decryption = true   # decrypt SecureString parameters
KeyTypeDescription
[providers.paths]table (name → path)Explicit plugin locations keyed by provider short name. Takes priority over discovered providers. Point at the directory holding manifest.toml or directly at the executable.

[provider_registry]

KeyTypeDefaultDescription
urlstringpublic registryIndex URL. Supports https://, file://, or a local path. The --registry flag overrides it.
allow_unsignedboolfalsePermit installing unsigned artifacts. By default every artifact's ed25519 signature is verified against the built-in trusted key.
install_scopestring"project""project" or "user". In the current core the effective scope comes from --project/--user.
public_keystring—Additional base64 ed25519 public key trusted in addition to the built-in release key (for a custom/private registry).

[resolution]

KeyTypeDefaultDescription
missingstring"error"Behavior when a referenced secret isn't found: error, warn, or ignore.
concurrencyinteger8Max concurrent provider resolutions.

[conflicts]

KeyTypeDefaultDescription
warn_on_any_shadowboolfalseWarn whenever any key is shadowed across sources.
error_on_shadowarray of string[]Treat shadowing of these specific keys as a hard error.

[sensitive]

KeyTypeDefaultDescription
keysarray of string[]Key names to redact in diagnostics (also matched as case-insensitive substrings).
patternsarray of string[]Glob patterns; any matching key is redacted.

Values resolved from remote references are always redacted, regardless of this list.

Complete annotated example 
default_profile = "dev"

[profile.dev]
env_file = ".env"
env_local_file = ".env.local"

[defaults]
LOG_LEVEL = "info"

[precedence]
order = ["cli", "system", "remote", "env.local", "env", "defaults"]

[providers.aws]
region = "us-east-1"

[provider_registry]
url = "https://geetchoubey.github.io/dotenv-cloud/index.json"
allow_unsigned = false

[resolution]
missing = "error"
concurrency = 8

[conflicts]
warn_on_any_shadow = false
error_on_shadow = []

[sensitive]
keys = ["DB_PASSWORD", "API_KEY"]
patterns = ["*_TOKEN", "*_SECRET"]

Global flags

These precede the subcommand and apply everywhere, e.g. dotenv-cloud --profile prod run -- ./app.

FlagArgumentDescription
--configpathUse an explicit config file (skips discovery).
--profilenameSelect a named profile.
--env-filepathOverride the profile's .env path.
--env-local-filepathOverride the profile's .env.local path.
--no-env-file—Do not load .env.
--no-env-local—Do not load .env.local.
--setKEY=VALUEAdd/override a value at CLI precedence. Repeatable.
--strict—Treat warnings as errors.
--timeoutdurationProvider request timeout, e.g. 2s, 500ms, 1500 (ms). Default 2s.
--no-color—Disable colored output.
--verbose—Emit diagnostic metadata (no secrets).
--quiet—Suppress non-error output.
-h, --help / -V, --version—Help / version.

Commands

init

Scan config and dotenv files, install required providers, write the lockfile.

FlagArgDescription
--yes—Actually install the missing providers (otherwise dry).
--upgrade—(Re)install all detected providers, not just missing ones.
--offline—Skip installation; report only.
--project / --user—Install scope (default project).
--registryurlOverride the registry index URL.
--lockfilepathLockfile path (default dotenv-cloud.lock).

run

Resolve the environment and execute a child process. The command follows --.

dotenv-cloud run --require DB_PASSWORD -- node server.js
FlagArgDescription
--clear-env—Start the child with only the resolved env, not the inherited system env.
--preserveVARSComma-separated system vars to keep when --clear-env is used.
--requireKEYFail if the key is absent after resolution. Repeatable.
--dry-run—Print a redacted summary and exit without running.
--redact-summary—Print a redacted source summary, then run.
--install-missing-providers—Install any missing providers before resolving.

export

Print shell-compatible assignments for the project environment (.env, .env.local, remote, [defaults], --set) — not your shell's ambient variables.

FlagArgDescription
--shell / --formatshellbash (default), zsh, fish, powershell (aliases pwsh, ps1).
--includeKEYOnly emit these keys. Repeatable.
--excludeKEYOmit these keys. Repeatable.
--no-comments—Omit the generated header comment.

build

Materialize the project environment (same sources as export; not ambient shell variables) to a file or stdout.

FlagArgDescription
--modedotenv|jsonOutput format (default dotenv).
--outputpathWrite to a file instead of stdout.
--force—Overwrite an existing output file.
--chmodmodeFile mode for the output (default 0600; Unix).
--include / --excludeKEYKey filters. Repeatable.

resolve

Resolve and print one key (redacted unless --show).

FlagArgDescription
<key>positionalThe key to resolve.
--show—Print the actual value instead of a redaction.
--json—JSON output.
--source—Include winning source and shadowed sources.

validate

Parse config, check references, optionally contact providers.

FlagArgDescription
--providers—Also perform a provider connectivity/handshake check.
--no-providers—Skip the provider check.
--all-profiles—Validate across all configured profiles.
--json—JSON output.

doctor

Diagnose local setup, credentials, config, and installed providers. No flags.

providers

Manage externally installed provider plugins.

SubcommandArgs & flagsDescription
list--jsonList installed providers.
search <query>--registrySearch the registry.
install <name>--registry --project --user --yes --allow-unsigned --jsonDownload, verify, install. Pin with name@version.
update [name]--registry --project --user --yes --allow-unsignedUpdate one provider, or all if omitted.
info <name>--jsonShow provider metadata, schemes, version, integrity.
remove <name>--yesRemove an installed provider.
Shared flagDescription
--registry <url>Override the registry index URL.
--project / --userInstall scope (default project).
--allow-unsignedInstall without a verified signature.
--yesSkip the confirmation prompt.

completions

Print a shell completion script. Homebrew installs these automatically, and the install script wires them into your shell; you can also enable them manually:

# bash / zsh — add to your rc file
eval "$(dotenv-cloud completions zsh)"

# fish
dotenv-cloud completions fish > ~/.config/fish/completions/dotenv-cloud.fish
ArgumentDescription
<shell>bash, zsh, fish, powershell, or elvish.

Maintainer tooling: dotenv-cloud sign reads the signing key from DOTENV_CLOUD_SIGNING_KEY; dotenv-cloud keygen generates a keypair.
